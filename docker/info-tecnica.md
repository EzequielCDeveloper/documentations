## Container:

In a container you run an app with all the dependencies, just using the kernel of the OS.
You can use a file .yml to share the configuration of this container and the other people can use the exact same configuration.

![diagram-docker-engine](/assets/diagram-docker-engine.png)

## image:

template with all the neccesary information for instance a container. the aplication, packages, librarys, commands, ports, etc.

Exists hubs for the docker files, an example is:
